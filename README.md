# Крипто-бот — безопасная версия

## Шаг 1 — обязательно отзови старый токен (сделай прямо сейчас)

1. Открой [@BotFather](https://t.me/BotFather)
2. `/mybots` → выбери своего бота → **API Token** → **Revoke current token**
3. Скопируй новый токен, он понадобится в шаге 3

Старый токен (`8027726911:...`) считай скомпрометированным — использовать его больше нельзя.

## Шаг 2 — почисти историю на GitHub

Токен и пароль остаются в старых коммитах, даже если убрать их из последней версии файла. Проще всего:

1. Сделай репозиторий **приватным** (Settings → Danger Zone → Change visibility), если он публичный
2. Или удали репозиторий и запушь заново с этой версией кода — так старая история с токеном исчезнет полностью

Для маленького проекта проще и надёжнее второй вариант.

## Шаг 3 — настрой переменные окружения

### Локально (для теста на своём компьютере)
```bash
cp .env.example .env
```
Открой `.env` и впиши:
- `BOT_TOKEN` — новый токен из шага 1
- `ADMIN_PASSWORD` — новый пароль (не используй старый "169169")
- `ADMIN_USERNAMES` — свои юзернеймы через запятую, без @

### На Railway
1. Открой свой проект на Railway → вкладка **Variables**
2. Добавь три переменные: `BOT_TOKEN`, `ADMIN_PASSWORD`, `ADMIN_USERNAMES` — с теми же значениями, что в `.env`
3. Redeploy — бот подхватит их автоматически, `.env` файл на сервере не нужен

## Установка и запуск локально
```bash
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
python bot.py
```

## Что изменилось в коде
- Токен, пароль админки и список админов теперь читаются из переменных окружения (`os.getenv`), а не хранятся в коде
- Добавлен `.gitignore`, чтобы `.env` никогда случайно не попал в git
- Если `BOT_TOKEN` или `ADMIN_PASSWORD` не заданы — бот сразу упадёт с понятной ошибкой вместо того, чтобы тихо работать в незащищённом виде

## Дальше
Когда перезальёшь на Railway с новыми переменными — напиши, и продолжим с портфель-трекером и persistent-хранилищем (SQLite), чтобы алерты и юзеры не слетали при каждом перезапуске.
